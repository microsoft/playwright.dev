Playwright can navigate to URLs and handle navigations caused by page interactions. This guide covers common scenarios to wait for page navigations and loading to complete.

- [Navigation lifecycle](#navigation-lifecycle)
- [Scenarios initiated by browser UI](#scenarios-initiated-by-browser-ui)
- [Scenarios initiated by page interaction](#scenarios-initiated-by-page-interaction)
- [Advanced patterns](#advanced-patterns)

## Navigation lifecycle

Playwright splits the process of showing a new document in a page into **navigation** and **loading**.

**Navigation starts** by changing the page URL or by interacting with the page (e.g., clicking a link). The navigation intent may be canceled, for example, on hitting an unresolved DNS address or transformed into a file download.

**Navigation is committed** when the response headers have been parsed and session history is updated. Only after the navigation succeeds (is committed), the page starts **loading** the document.

**Loading** covers getting the remaining response body over the network, parsing, executing the scripts and firing load events:
- [page.url](/api/class-page.mdx#page-url) is set to the new url
- document content is loaded over network and parsed
- [page.on("domcontentloaded")](/api/class-page.mdx#page-event-dom-content-loaded) event is fired
- page executes some scripts and loads resources like stylesheets and images
- [page.on("load")](/api/class-page.mdx#page-event-load) event is fired
- page executes dynamically loaded scripts
- `networkidle` is fired when no new network requests are made for 500 ms

## Scenarios initiated by browser UI

Navigations can be initiated by changing the URL bar, reloading the page or going back or forward in session history.

### Auto-wait

Navigating to a URL auto-waits for the page to fire the `load` event. If the page does a client-side redirect before `load`, [page.goto(url, **kwargs)](/api/class-page.mdx#page-goto) will auto-wait for the redirected page to fire the `load` event.

  defaultValue="sync"
  values={[

    {label: 'Async', value: 'async'}
  ]
}>
<TabItem value="sync">

```py
# Navigate the page
page.goto("https://example.com")
```

</TabItem>
<TabItem value="async">

```py
# Navigate the page
await page.goto("https://example.com")
```

</TabItem>
</Tabs>

### Custom wait

Override the default behavior to wait until a specific event, like `networkidle`.

<Tabs

```

</TabItem>

```

### Wait for element

In lazy-loaded pages, it can be useful to wait until an element is visible with [locator.wait_for(**kwargs)](/api/class-locator.mdx#locator-wait-for). Alternatively, page interactions like [page.click(selector, **kwargs)](/api/class-page.mdx#page-click) auto-wait for elements.

<Tabs
  groupId="python-flavor"
  defaultValue="sync"
  values={[
    {label: 'Sync', value: 'sync'},
    {label: 'Async', value: 'async'}
  ]
}>
<TabItem value="sync">

```py
# Navigate and wait for element
page.goto("https://example.com")
page.locator("text=example domain").wait_for()

# Navigate and click element
# Click will auto-wait for the element
page.goto("https://example.com")
page.locator("text=example domain").click()
```

</TabItem>
<TabItem value="async">

```py
# Navigate and wait for element
await page.goto("https://example.com")
await page.locator("text=example domain").wait_for()

# Navigate and click element
# Click will auto-wait for the element
await page.goto("https://example.com")
await page.locator("text=example domain").click()
```

</TabItem>
</Tabs>

## Scenarios initiated by page interaction

In the scenarios below, [locator.click(**kwargs)](/api/class-locator.mdx#locator-click) initiates a navigation and then waits for the navigation to complete.

### Auto-wait

By default, [locator.click(**kwargs)](/api/class-locator.mdx#locator-click) will wait for the navigation step to complete. This can be combined with a page interaction on the navigated page which would auto-wait for an element.

<Tabs
  groupId="python-flavor"
  defaultValue="sync"
  values={[
    {label: 'Sync', value: 'sync'},
    {label: 'Async', value: 'async'}
  ]
}>
<TabItem value="sync">

```py
# Click will auto-wait for navigation to complete
page.locator("text=Login").click()

# Fill will auto-wait for element on navigated page
page.locator("#username").fill("John Doe")
```

</TabItem>
<TabItem value="async">

```py
# Click will auto-wait for navigation to complete
await page.locator("text=Login").click()

# Fill will auto-wait for element on navigated page
await page.locator("#username").fill("John Doe")
```

</TabItem>
</Tabs>

### Custom wait

`locator.click` can be combined with [page.wait_for_load_state(**kwargs)](/api/class-page.mdx#page-wait-for-load-state) to wait for a loading event.

<Tabs
  groupId="python-flavor"
  defaultValue="sync"
  values={[
    {label: 'Sync', value: 'sync'},
    {label: 'Async', value: 'async'}
  ]
}>
<TabItem value="sync">

```py
page.locator("button").click(); # Click triggers navigation
page.wait_for_load_state("networkidle"); # This waits for the "networkidle"
```

</TabItem>
<TabItem value="async">

```py
await page.locator("button").click(); # Click triggers navigation
await page.wait_for_load_state("networkidle"); # This waits for the "networkidle"
```

</TabItem>
</Tabs>

### Wait for element

In lazy-loaded pages, it can be useful to wait until an element is visible with [locator.wait_for(**kwargs)](/api/class-locator.mdx#locator-wait-for). Alternatively, page interactions like [locator.click(**kwargs)](/api/class-locator.mdx#locator-click) auto-wait for elements.

<Tabs
  groupId="python-flavor"
  defaultValue="sync"
  values={[
    {label: 'Sync', value: 'sync'},
    {label: 'Async', value: 'async'}
  ]
}>
<TabItem value="sync">

```py
# Click triggers navigation
page.locator("text=Login").click()
# Click will auto-wait for the element
page.locator("#username").wait_for()

# Click triggers navigation
page.locator("text=Login").click()
# Fill will auto-wait for element
page.locator("#username").fill("John Doe")
```

</TabItem>
<TabItem value="async">

```py
# Click will auto-wait for the element and trigger navigation
await page.locator("text=Login").click()
# Wait for the element
await page.locator("#username").wait_for()

# Click triggers navigation
await page.locator("text=Login").click()
# Fill will auto-wait for element
await page.locator("#username").fill("John Doe")
```

</TabItem>
</Tabs>

### Asynchronous navigation

Clicking an element could trigger asynchronous processing before initiating the navigation. In these cases, it is recommended to explicitly call [page.expect_navigation(**kwargs)](/api/class-page.mdx#page-wait-for-navigation). For example:
* Navigation is triggered from a `setTimeout`
* Page waits for network requests before navigation

<Tabs
  groupId="python-flavor"
  defaultValue="sync"
  values={[
    {label: 'Sync', value: 'sync'},
    {label: 'Async', value: 'async'}
  ]
}>
<TabItem value="sync">

```py
# Waits for the next navigation. Using Python context manager
# prevents a race condition between clicking and waiting for a navigation.
with page.expect_navigation():
    # Triggers a navigation after a timeout
    page.locator("a").click()
```

</TabItem>
<TabItem value="async">

```py
# Waits for the next navigation. Using Python context manager
# prevents a race condition between clicking and waiting for a navigation.
async with page.expect_navigation():
    # Triggers a navigation after a timeout
    await page.locator("div.delayed-navigation").click()
```

</TabItem>
</Tabs>

### Multiple navigations

Clicking an element could trigger multiple navigations. In these cases, it is recommended to explicitly [page.expect_navigation(**kwargs)](/api/class-page.mdx#page-wait-for-navigation) to a specific url. For example:
* Client-side redirects issued after the `load` event
* Multiple pushes to history state

<Tabs
  groupId="python-flavor"
  defaultValue="sync"
  values={[
    {label: 'Sync', value: 'sync'},
    {label: 'Async', value: 'async'}
  ]
}>
<TabItem value="sync">

```py
# Using Python context manager prevents a race condition
# between clicking and waiting for a navigation.
with page.expect_navigation(url="**/login"):
    # Triggers a navigation with a script redirect
    page.locator("a").click()
```

</TabItem>
<TabItem value="async">

```py
# Using Python context manager prevents a race condition
# between clicking and waiting for a navigation.
async with page.expect_navigation(url="**/login"):
    # Triggers a navigation with a script redirect
    await page.locator("a").click()
```

</TabItem>
</Tabs>

### Loading a popup

When popup is opened, explicitly calling [page.wait_for_load_state(**kwargs)](/api/class-page.mdx#page-wait-for-load-state) ensures that popup is loaded to the desired state.

<Tabs
  groupId="python-flavor"
  defaultValue="sync"
  values={[
    {label: 'Sync', value: 'sync'},
    {label: 'Async', value: 'async'}
  ]
}>
<TabItem value="sync">

```py
with page.expect_popup() as popup_info:
    page.locator('a[target="_blank"]').click() # Opens popup
popup = popup_info.value
popup.wait_for_load_state("load")
```

</TabItem>
<TabItem value="async">

```py
async with page.expect_popup() as popup_info:
    await page.locator('a[target="_blank"]').click() # Opens popup
popup = await popup_info.value
await popup.wait_for_load_state("load")
```

</TabItem>
</Tabs>

## Advanced patterns

For pages that have complicated loading patterns, [page.wait_for_function(expression, **kwargs)](/api/class-page.mdx#page-wait-for-function) is a powerful and extensible approach to define a custom wait criteria.

<Tabs
  groupId="python-flavor"
  defaultValue="sync"
  values={[
    {label: 'Sync', value: 'sync'},
    {label: 'Async', value: 'async'}
  ]
}>
<TabItem value="sync">

```py
page.goto("http://example.com")
page.wait_for_function("() => window.amILoadedYet()")
# Ready to take a screenshot, according to the page itself.
page.screenshot()
```

</TabItem>
<TabItem value="async">

```py
await page.goto("http://example.com")
await page.wait_for_function("() => window.amILoadedYet()")
# Ready to take a screenshot, according to the page itself.
await page.screenshot()
```

</TabItem>
</Tabs>
