### header_value {#response-header-value}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.15</font><x-search>response.header_value</x-search>

Returns the value of the header matching the name. The name is case insensitive. If multiple headers have the same name (except `set-cookie`), they are returned as a list separated by `, `. For `set-cookie`, the `\n` separator is used. If no headers are found, `null` is returned.

**Usage**

```python
response.header_value(name)
```

**Arguments**
- `name` [str]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="response-header-value-option-name"/><a href="#response-header-value-option-name" class="list-anchor">#</a>
  
  Name of the header.

**Returns**
- [NoneType]|[str]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="response-header-value-return"/><a href="#response-header-value-return" class="list-anchor">#</a>

### headers_array {#response-headers-array}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.15</font><x-search>response.headers_array</x-search>

An array with all the request HTTP headers associated with this response. Unlike [response.all_headers()](/api/class-response.mdx#response-all-headers), header names are NOT lower-cased. Headers with multiple entries, such as `Set-Cookie`, appear in the array multiple times.

**Usage**

```python
response.headers_array()
```

**Returns**
- [List]\[[Dict]\]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="response-headers-array-return"/><a href="#response-headers-array-return" class="list-anchor">#</a>
  - `name` [str]
    
    Name of the header.
  - `value` [str]
    
    Value of the header.

### security_details {#response-security-details}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.13</font><x-search>response.security_details</x-search>

Returns SSL and other security information.

**Usage**

```python
response.security_details()
```

**Returns**
- [NoneType]|[Dict]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="response-security-details-return"/><a href="#response-security-details-return" class="list-anchor">#</a>
  - `issuer` [str] *(optional)*
    
    Common Name component of the Issuer field. from the certificate. This should only be used for informational purposes. Optional.
  - `protocol` [str] *(optional)*
    
    The specific TLS protocol used. (e.g. `TLS 1.3`). Optional.
  - `subjectName` [str] *(optional)*
    
    Common Name component of the Subject field from the certificate. This should only be used for informational purposes. Optional.
  - `validFrom` [float] *(optional)*
    
    Unix timestamp (in seconds) specifying when this cert becomes valid. Optional.
  - `validTo` [float] *(optional)*
    
    Unix timestamp (in seconds) specifying when this cert becomes invalid. Optional.

### headers {#response-headers}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.8</font><x-search>response.headers</x-search>

An object with the response HTTP headers. The header names are lower-cased. Note that this method does not return security-related headers, including cookie-related ones. You can use [response.all_headers()](/api/class-response.mdx#response-all-headers) for complete list of headers that include `cookie` information.

```python
response.headers
```
