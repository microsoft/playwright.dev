Playwright can interact with the web page dialogs such as [`alert`](https://developer.mozilla.org/en-US/docs/Web/API/Window/alert), [`confirm`](https://developer.mozilla.org/en-US/docs/Web/API/Window/confirm), [`prompt`](https://developer.mozilla.org/en-US/docs/Web/API/Window/prompt) as well as [`beforeunload`](https://developer.mozilla.org/en-US/docs/Web/API/Window/beforeunload_event) confirmation.

- [alert(), confirm(), prompt() dialogs](#alert-confirm-prompt-dialogs)
- [beforeunload dialog](#beforeunload-dialog)

## alert(), confirm(), prompt() dialogs

By default, dialogs are auto-dismissed by Playwright, so you don't have to handle them. However, you can register a dialog handler before the action that triggers the dialog to accept or decline it.

<Tabs
  groupId="python-flavor"
  defaultValue="sync"
  values={[
    {label: 'Sync', value: 'sync'},
    {label: 'Async', value: 'async'}
  ]
}>
<TabItem value="sync">

```py
page.on("dialog", lambda dialog: dialog.accept())
page.click("button")
```

</TabItem>
<TabItem value="async">

```py
page.on("dialog", lambda dialog: dialog.accept())
await page.click("button")
```

</TabItem>
</Tabs>

:::note
[page.on("dialog")](./api/class-page.mdx#pageondialog) listener **must handle** the dialog. Otherwise your action will stall, be it [page.click(selector, **kwargs)](./api/class-page.mdx#pageclickselector-kwargs), [page.evaluate(expression, **kwargs)](./api/class-page.mdx#pageevaluateexpression-kwargs) or any other. That's because dialogs in Web are modal and block further page execution until they are handled.
:::

As a result, following snippet will never resolve:

:::warning
WRONG!
:::

<Tabs
  groupId="python-flavor"
  defaultValue="sync"
  values={[
    {label: 'Sync', value: 'sync'},
    {label: 'Async', value: 'async'}
  ]
}>
<TabItem value="sync">

```py
page.on("dialog", lambda dialog: print(dialog.message))
page.click("button") # Will hang here
```

</TabItem>
<TabItem value="async">

```py
page.on("dialog", lambda dialog: print(dialog.message))
await page.click("button") # Will hang here
```

</TabItem>
</Tabs>

:::note
If there is no listener for [page.on("dialog")](./api/class-page.mdx#pageondialog), all dialogs are automatically dismissed.
:::

## beforeunload dialog

When [page.close(**kwargs)](./api/class-page.mdx#pageclosekwargs) is invoked with the truthy `run_before_unload` value, it page runs its unload handlers. This is the only case when [page.close(**kwargs)](./api/class-page.mdx#pageclosekwargs) does not wait for the page to actually close, because it might be that the page stays open in the end of the operation.

You can register a dialog handler to handle the beforeunload dialog yourself:

<Tabs
  groupId="python-flavor"
  defaultValue="sync"
  values={[
    {label: 'Sync', value: 'sync'},
    {label: 'Async', value: 'async'}
  ]
}>
<TabItem value="sync">

```py
def handle_dialog(dialog):
    assert dialog.type == 'beforeunload'
    dialog.dismiss()

page.on('dialog', lambda: handle_dialog)
page.close(run_before_unload=True)
```

</TabItem>
<TabItem value="async">

```py
async def handle_dialog(dialog):
    assert dialog.type == 'beforeunload'
    await dialog.dismiss()

page.on('dialog', lambda: handle_dialog)
await page.close(run_before_unload=True)
```

</TabItem>
</Tabs>
