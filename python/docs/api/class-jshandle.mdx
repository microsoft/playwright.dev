JSHandle represents an in-page JavaScript object. JSHandles can be created with the [page.evaluate_handle()](/api/class-page.mdx#page-evaluate-handle) method.

<Tabs
  groupId="python-flavor"
  defaultValue="sync"
  values={[
    {label: 'Sync', value: 'sync'},
    {label: 'Async', value: 'async'}
  ]
}>
<TabItem value="sync">

```py
window_handle = page.evaluate_handle("window")
# ...
```

</TabItem>
<TabItem value="async">

```py
window_handle = await page.evaluate_handle("window")
# ...
```

</TabItem>
</Tabs>

JSHandle prevents the referenced JavaScript object being garbage collected unless the handle is exposed with [js_handle.dispose()](/api/class-jshandle.mdx#js-handle-dispose). JSHandles are auto-disposed when their origin frame gets navigated or the parent context gets destroyed.

JSHandle instances can be used as an argument in [page.eval_on_selector()](/api/class-page.mdx#page-eval-on-selector), [page.evaluate()](/api/class-page.mdx#page-evaluate) and [page.evaluate_handle()](/api/class-page.mdx#page-evaluate-handle) methods.

### evaluate {#js-handle-evaluate}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.8</font>


Returns the return value of `expression`.

This method passes this handle as the first argument to `expression`.

If `expression` returns a [Promise], then `handle.evaluate` would wait for the promise to resolve and return its value.

**Usage**

<Tabs
  groupId="python-flavor"
  defaultValue="sync"
  values={[
    {label: 'Sync', value: 'sync'},
    {label: 'Async', value: 'async'}
  ]
}>
<TabItem value="sync">

```py
tweet_handle = page.query_selector(".tweet .retweets")
assert tweet_handle.evaluate("node => node.innerText") == "10 retweets"
```

</TabItem>
<TabItem value="async">

```py
tweet_handle = await page.query_selector(".tweet .retweets")
assert await tweet_handle.evaluate("node => node.innerText") == "10 retweets"
```

</TabItem>
</Tabs>

**Parameters**
- `expression` &#60;[str]&#62; JavaScript expression to be evaluated in the browser context. If the expression evaluates to a function, the function is automatically invoked.<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="js-handle-evaluate-option-expression"/><a href="#js-handle-evaluate-option-expression" class="list-anchor">#</a>
- `arg` &#60;[EvaluationArgument]&#62; Optional argument to pass to `expression`.<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="js-handle-evaluate-option-arg"/><a href="#js-handle-evaluate-option-arg" class="list-anchor">#</a>

### evaluate_handle {#js-handle-evaluate-handle}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.8</font>


Returns the return value of `expression` as a [JSHandle].

This method passes this handle as the first argument to `expression`.

The only difference between `jsHandle.evaluate` and `jsHandle.evaluateHandle` is that `jsHandle.evaluateHandle` returns [JSHandle].

If the function passed to the `jsHandle.evaluateHandle` returns a [Promise], then `jsHandle.evaluateHandle` would wait for the promise to resolve and return its value.

See [page.evaluate_handle()](/api/class-page.mdx#page-evaluate-handle) for more details.

**Usage**

```python
js_handle.evaluate_handle(expression)
js_handle.evaluate_handle(expression, **kwargs)
```

**Parameters**
- `expression` &#60;[str]&#62; JavaScript expression to be evaluated in the browser context. If the expression evaluates to a function, the function is automatically invoked.<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="js-handle-evaluate-handle-option-expression"/><a href="#js-handle-evaluate-handle-option-expression" class="list-anchor">#</a>
- `arg` &#60;[EvaluationArgument]&#62; Optional argument to pass to `expression`.<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="js-handle-evaluate-handle-option-arg"/><a href="#js-handle-evaluate-handle-option-arg" class="list-anchor">#</a>

**Returns**
- &#60;[JSHandle]&#62;<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="js-handle-evaluate-handle-return"/><a href="#js-handle-evaluate-handle-return" class="list-anchor">#</a>

### get_properties {#js-handle-get-properties}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.8</font>


The method returns a map with **own property names** as keys and JSHandle instances for the property values.

**Usage**

<Tabs
  groupId="python-flavor"
  defaultValue="sync"
  values={[
    {label: 'Sync', value: 'sync'},
    {label: 'Async', value: 'async'}
  ]
}>
<TabItem value="sync">

```py
handle = page.evaluate_handle("({window, document})")
properties = handle.get_properties()
window_handle = properties.get("window")
document_handle = properties.get("document")
handle.dispose()
```

</TabItem>
<TabItem value="async">

```py
handle = await page.evaluate_handle("({window, document})")
properties = await handle.get_properties()
window_handle = properties.get("window")
document_handle = properties.get("document")
await handle.dispose()
```

</TabItem>
</Tabs>

**Returns**
- &#60;[Map]\[[str], [JSHandle]\]&#62;<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="js-handle-get-properties-return"/><a href="#js-handle-get-properties-return" class="list-anchor">#</a>

### json_value {#js-handle-json-value}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.8</font>


Returns a JSON representation of the object. If the object has a `toJSON` function, it **will not be called**.

:::note
The method will return an empty JSON object if the referenced object is not stringifiable. It will throw an error if the object has circular references.
:::

**Usage**

```python
js_handle.json_value()
```

- &#60;[Serializable]&#62;<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="js-handle-json-value-return"/><a href="#js-handle-json-value-return" class="list-anchor">#</a>
