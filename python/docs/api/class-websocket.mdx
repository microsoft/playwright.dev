## expect_event {#web-socket-wait-for-event}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.8</font>

Waits for event to fire and passes its value into the predicate function. Returns when the predicate returns truthy value. Will throw an error if the webSocket is closed before the event is fired. Returns the event data value.

#### Usage

```python
web_socket.expect_event(event)
web_socket.expect_event(event, **kwargs)
```

#### Parameters
- `event` &#60;[str]&#62; Event name, same one would pass into `webSocket.on(event)`.<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="web-socket-wait-for-event-option-event"/><a href="#web-socket-wait-for-event-option-event" class="list-anchor">#</a>
- `predicate` &#60;[Callable]&#62; Receives the event data and resolves to truthy value when the waiting should resolve.<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="web-socket-wait-for-event-option-predicate"/><a href="#web-socket-wait-for-event-option-predicate" class="list-anchor">#</a>
- `timeout` &#60;[float]&#62; Maximum time to wait for in milliseconds. Defaults to `30000` (30 seconds). Pass `0` to disable timeout. The default value can be changed by using the [browser_context.set_default_timeout()](/api/class-browsercontext.mdx#browser-context-set-default-timeout).<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="web-socket-wait-for-event-option-timeout"/><a href="#web-socket-wait-for-event-option-timeout" class="list-anchor">#</a>

#### Returns
-  &#60;[EventContextManager]&#62;<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="web-socket-wait-for-event-return"/><a href="#web-socket-wait-for-event-return" class="list-anchor">#</a>

## wait_for_event {#web-socket-wait-for-event-2}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.8</font>

:::note
In most cases, you should use [web_socket.expect_event()](/api/class-websocket.mdx#web-socket-wait-for-event).
:::

Waits for given `event` to fire. If predicate is provided, it passes event's value into the `predicate` function and waits for `predicate(event)` to return a truthy value. Will throw an error if the socket is closed before the `event` is fired.

#### Usage

```python
web_socket.wait_for_event(event)
web_socket.wait_for_event(event, **kwargs)
```

#### Parameters
- `event` &#60;[str]&#62; Event name, same one typically passed into `*.on(event)`.<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="web-socket-wait-for-event-2-option-event"/><a href="#web-socket-wait-for-event-2-option-event" class="list-anchor">#</a>
- `predicate` &#60;[Callable]&#62; Receives the event data and resolves to truthy value when the waiting should resolve.<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="web-socket-wait-for-event-2-option-predicate"/><a href="#web-socket-wait-for-event-2-option-predicate" class="list-anchor">#</a>
- `timeout` &#60;[float]&#62; Maximum time to wait for in milliseconds. Defaults to `30000` (30 seconds). Pass `0` to disable timeout. The default value can be changed by using the [browser_context.set_default_timeout()](/api/class-browsercontext.mdx#browser-context-set-default-timeout).<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="web-socket-wait-for-event-2-option-timeout"/><a href="#web-socket-wait-for-event-2-option-timeout" class="list-anchor">#</a>

#### Returns
-  &#60;[Any]&#62;<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="web-socket-wait-for-event-2-return"/><a href="#web-socket-wait-for-event-2-return" class="list-anchor">#</a>
