The Touchscreen class operates in main-frame CSS pixels relative to the top-left corner of the viewport. Methods on the touchscreen can only be used in browser contexts that have been initialized with `hasTouch` set to true.

This class is limited to emulating tap gestures. For examples of other gestures simulated by manually dispatching touch events, see the [emulating legacy touch events](../touch-events.mdx) page.


---

## Methods

### tap {#touchscreen-tap}

<font size="2" style={{position: "relative", top: "-20px"}}>Added before v1.9</font><x-search>touchscreen.tap</x-search>

Dispatches a `touchstart` and `touchend` event with a single touch at the position ([x](/api/class-touchscreen.mdx#touchscreen-tap-option-x),[y](/api/class-touchscreen.mdx#touchscreen-tap-option-y)).

:::note

[page.tap()](/api/class-page.mdx#page-tap) the method will throw if [has_touch](/api/class-browser.mdx#browser-new-context-option-has-touch) option of the browser context is false.
:::

**Usage**

```python
touchscreen.tap(x, y)
```

**Arguments**
- `x` [float]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="touchscreen-tap-option-x"/><a href="#touchscreen-tap-option-x" class="list-anchor">#</a>
  
  X coordinate relative to the main frame's viewport in CSS pixels.
- `y` [float]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="touchscreen-tap-option-y"/><a href="#touchscreen-tap-option-y" class="list-anchor">#</a>
  
  Y coordinate relative to the main frame's viewport in CSS pixels.

**Returns**
- [NoneType]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="touchscreen-tap-return"/><a href="#touchscreen-tap-return" class="list-anchor">#</a>
