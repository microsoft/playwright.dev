Playwright Test provides many options to configure test environment, [Browser], [BrowserContext] and more.

These options are usually provided in the [configuration file](../test-configuration.mdx) through [testConfig.use](/api/class-testconfig.mdx#test-config-use) and [testProject.use](/api/class-testproject.mdx#test-project-use).

```js

const config: PlaywrightTestConfig = {
  use: {
    headless: false,
    viewport: { width: 1280, height: 720 },

    video: 'on-first-retry',
  },
};
export default config;
```

## testOptions.acceptDownloads {#test-options-accept-downloads}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[boolean]&#62;

Whether to automatically download all the attachments. Defaults to `true` where all the downloads are accepted.

## testOptions.actionTimeout {#test-options-action-timeout}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[number]&#62;

Default timeout for each Playwright action in milliseconds, defaults to 0 (no timeout).

This is a default timeout for all Playwright actions, same as configured via [page.setDefaultTimeout(timeout)](/api/class-page.mdx#page-set-default-timeout).

Learn more about [various timeouts](../test-timeouts.mdx).

## testOptions.baseURL {#test-options-base-url}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[string]&#62;

When using [page.goto(url[, options])](/api/class-page.mdx#page-goto), [page.route(url, handler[, options])](/api/class-page.mdx#page-route), [page.waitForURL(url[, options])](/api/class-page.mdx#page-wait-for-url), [page.waitForRequest(urlOrPredicate[, options])](/api/class-page.mdx#page-wait-for-request), or [page.waitForResponse(urlOrPredicate[, options])](/api/class-page.mdx#page-wait-for-response) it takes the base URL in consideration by using the [`URL()`](https://developer.mozilla.org/en-US/docs/Web/API/URL/URL) constructor for building the corresponding URL. Examples:
* baseURL: `http://localhost:3000` and navigating to `/bar.html` results in `http://localhost:3000/bar.html`
* baseURL: `http://localhost:3000/foo/` and navigating to `./bar.html` results in `http://localhost:3000/foo/bar.html`
* baseURL: `http://localhost:3000/foo` (without trailing slash) and navigating to `./bar.html` results in `http://localhost:3000/bar.html`

## testOptions.browserName {#test-options-browser-name}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;"chromium"|"firefox"|"webkit"&#62;

Name of the browser that runs tests. Defaults to `'chromium'`. Most of the time you should set `browserName` in your [TestConfig]:

## testOptions.bypassCSP {#test-options-bypass-csp}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[boolean]&#62;

Toggles bypassing page's Content-Security-Policy.

## testOptions.channel {#test-options-channel}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[string]&#62;

Browser distribution channel.  Supported values are "chrome", "chrome-beta", "chrome-dev", "chrome-canary", "msedge", "msedge-beta", "msedge-dev", "msedge-canary". Read more about using [Google Chrome and Microsoft Edge](../browsers.mdx#google-chrome--microsoft-edge).

## testOptions.colorScheme {#test-options-color-scheme}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;"light"|"dark"|"no-preference"&#62;

Emulates `'prefers-colors-scheme'` media feature, supported values are `'light'`, `'dark'`, `'no-preference'`. See [page.emulateMedia([options])](/api/class-page.mdx#page-emulate-media) for more details. Defaults to `'light'`.

## testOptions.connectOptions {#test-options-connect-options}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[void]|[Object]&#62;
  - `wsEndpoint` &#60;[string]&#62; A browser websocket endpoint to connect to.
  - `headers?` &#60;[void]|[Object]<[string], [string]>&#62; Additional HTTP headers to be sent with web socket connect request. Optional.
  - `timeout?` &#60;[number]&#62; Timeout in milliseconds for the connection to be established. Optional, defaults to no timeout.

When connect options are specified, default [fixtures.browser](/api/class-fixtures.mdx#fixtures-browser), [fixtures.context](/api/class-fixtures.mdx#fixtures-context) and [fixtures.page](/api/class-fixtures.mdx#fixtures-page) use the remote browser instead of launching a browser locally, and any launch options like [testOptions.headless](/api/class-testoptions.mdx#test-options-headless) or [testOptions.channel](/api/class-testoptions.mdx#test-options-channel) are ignored.

## testOptions.contextOptions {#test-options-context-options}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[Object]&#62;

Options used to create the context, as passed to [browser.newContext([options])](/api/class-browser.mdx#browser-new-context). Specific options like [testOptions.viewport](/api/class-testoptions.mdx#test-options-viewport) take priority over this.

## testOptions.deviceScaleFactor {#test-options-device-scale-factor}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[number]&#62;

Specify device scale factor (can be thought of as dpr). Defaults to `1`.

## testOptions.extraHTTPHeaders {#test-options-extra-http-headers}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[Object]<[string], [string]>&#62;

An object containing additional HTTP headers to be sent with every request.

## testOptions.geolocation {#test-options-geolocation}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[Object]&#62;
  - `latitude` &#60;[number]&#62; Latitude between -90 and 90.
  - `longitude` &#60;[number]&#62; Longitude between -180 and 180.
  - `accuracy?` &#60;[number]&#62; Non-negative accuracy value. Defaults to `0`.

## testOptions.hasTouch {#test-options-has-touch}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[boolean]&#62;

Specifies if viewport supports touch events. Defaults to false.

## testOptions.headless {#test-options-headless}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[boolean]&#62;

Whether to run browser in headless mode. More details for [Chromium](https://developers.google.com/web/updates/2017/04/headless-chrome) and [Firefox](https://developer.mozilla.org/en-US/docs/Mozilla/Firefox/Headless_mode). Defaults to `true` unless the `devtools` option is `true`.

## testOptions.httpCredentials {#test-options-http-credentials}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[Object]&#62;
  - `username` &#60;[string]&#62;
  - `password` &#60;[string]&#62;

Credentials for [HTTP authentication](https://developer.mozilla.org/en-US/docs/Web/HTTP/Authentication).

## testOptions.ignoreHTTPSErrors {#test-options-ignore-https-errors}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[boolean]&#62;

Whether to ignore HTTPS errors when sending network requests. Defaults to `false`.

## testOptions.isMobile {#test-options-is-mobile}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[boolean]&#62;

Whether the `meta viewport` tag is taken into account and touch events are enabled. Defaults to `false`. Not supported in Firefox.

## testOptions.javaScriptEnabled {#test-options-java-script-enabled}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[boolean]&#62;

Whether or not to enable JavaScript in the context. Defaults to `true`.

## testOptions.launchOptions {#test-options-launch-options}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[Object]&#62;

Options used to launch the browser, as passed to [browserType.launch([options])](/api/class-browsertype.mdx#browser-type-launch). Specific options [testOptions.headless](/api/class-testoptions.mdx#test-options-headless) and [testOptions.channel](/api/class-testoptions.mdx#test-options-channel) take priority over this.

## testOptions.locale {#test-options-locale}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[string]&#62;

Specify user locale, for example `en-GB`, `de-DE`, etc. Locale will affect `navigator.language` value, `Accept-Language` request header value as well as number and date formatting rules.

## testOptions.navigationTimeout {#test-options-navigation-timeout}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[number]&#62;

Timeout for each navigation action in milliseconds. Defaults to 0 (no timeout).

This is a default navigation timeout, same as configured via [page.setDefaultNavigationTimeout(timeout)](/api/class-page.mdx#page-set-default-navigation-timeout).

Learn more about [various timeouts](../test-timeouts.mdx).

## testOptions.offline {#test-options-offline}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[boolean]&#62;

Whether to emulate network being offline. Defaults to `false`.

## testOptions.permissions {#test-options-permissions}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[Array]<[string]>&#62;

A list of permissions to grant to all pages in this context. See [browserContext.grantPermissions(permissions[, options])](/api/class-browsercontext.mdx#browser-context-grant-permissions) for more details.

## testOptions.proxy {#test-options-proxy}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[Object]&#62;
  - `server` &#60;[string]&#62; Proxy to be used for all requests. HTTP and SOCKS proxies are supported, for example `http://myproxy.com:3128` or `socks5://myproxy.com:3128`. Short form `myproxy.com:3128` is considered an HTTP proxy.
  - `bypass?` &#60;[string]&#62; Optional comma-separated domains to bypass proxy, for example `".com, chromium.org, .domain.com"`.
  - `username?` &#60;[string]&#62; Optional username to use if HTTP proxy requires authentication.
  - `password?` &#60;[string]&#62; Optional password to use if HTTP proxy requires authentication.

Network proxy settings.

## testOptions.screenshot {#test-options-screenshot}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;"off"|"on"|"only-on-failure"&#62;

Whether to automatically capture a screenshot after each test. Defaults to `'off'`.
* `'off'`: Do not capture screenshots.
* `'on'`: Capture screenshot after each test.
* `'only-on-failure'`: Capture screenshot after each test failure.

Learn more about [automatic screenshots](../test-configuration.mdx#automatic-screenshots).

## testOptions.serviceWorkers {#test-options-service-workers}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;"allow"|"block"&#62;

Whether to allow sites to register Service workers. Defaults to `'allow'`.
* `'allow'`: [Service Workers](https://developer.mozilla.org/en-US/docs/Web/API/Service_Worker_API) can be registered.
* `'block'`: Playwright will block all registration of Service Workers.

## testOptions.trace {#test-options-trace}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[Object]|"off"|"on"|"retain-on-failure"|"on-first-retry"&#62;
  - `mode` &#60;"off"|"on"|"retain-on-failure"|"on-first-retry"&#62; Trace recording mode.
  - `screenshots?` &#60;[boolean]&#62; Whether to capture screenshots during tracing. Screenshots are used to build a timeline preview. Defaults to true. Optional.
  - `snapshots?` &#60;[boolean]&#62; Whether to capture DOM snapshot on every action. Defaults to true. Optional.
  - `sources?` &#60;[boolean]&#62; Whether to include source files for trace actions. Defaults to true. Optional.

Whether to record trace for each test. Defaults to `'off'`.
* `'off'`: Do not record trace.
* `'on'`: Record trace for each test.
* `'retain-on-failure'`: Record trace for each test, but remove all traces from successful test runs.
* `'on-first-retry'`: Record trace only when retrying a test for the first time.

For more control, pass an object that specifies `mode` and trace features to enable.

Learn more about [recording trace](../test-configuration.mdx#record-test-trace).

## testOptions.video {#test-options-video}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[Object]|"off"|"on"|"retain-on-failure"|"on-first-retry"&#62;
  - `mode` &#60;"off"|"on"|"retain-on-failure"|"on-first-retry"&#62; Video recording mode.
  - `size?` &#60;[Object]&#62; Size of the recorded video. Optional.
    - `width` &#60;[number]&#62;
    - `height` &#60;[number]&#62;

Whether to record video for each test. Defaults to `'off'`.
* `'off'`: Do not record video.
* `'on'`: Record video for each test.
* `'retain-on-failure'`: Record video for each test, but remove all videos from successful test runs.
* `'on-first-retry'`: Record video only when retrying a test for the first time.

To control video size, pass an object with `mode` and `size` properties. If video size is not specified, it will be equal to [testOptions.viewport](/api/class-testoptions.mdx#test-options-viewport) scaled down to fit into 800x800. If `viewport` is not configured explicitly the video size defaults to 800x450. Actual picture of each page will be scaled down if necessary to fit the specified size.

Learn more about [recording video](../test-configuration.mdx#record-video).

## testOptions.viewport {#test-options-viewport}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[null]|[Object]&#62;
  - `width` &#60;[number]&#62; page width in pixels.
  - `height` &#60;[number]&#62; page height in pixels.

Emulates consistent viewport for each page. Defaults to an 1280x720 viewport. `null` disables the default viewport.
