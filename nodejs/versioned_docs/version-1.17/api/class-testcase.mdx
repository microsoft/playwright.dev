`TestCase` corresponds to every [test(title, testFunction)](./api/class-test.mdx#test-call) call in a test file. When a single [test(title, testFunction)](./api/class-test.mdx#test-call) is running in multiple projects or repeated multiple times, it will have multiple `TestCase` objects in corresponding projects' suites.


- [testCase.ok()](./api/class-testcase.mdx#test-case-ok)
- [testCase.outcome()](./api/class-testcase.mdx#test-case-outcome)
- [testCase.titlePath()](./api/class-testcase.mdx#test-case-title-path)
- [testCase.annotations](./api/class-testcase.mdx#test-case-annotations)
- [testCase.expectedStatus](./api/class-testcase.mdx#test-case-expected-status)

## testCase.ok() {#test-case-ok}
- returns:<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="test-case-ok-return"/> &#60;[boolean]&#62;<a href="#test-case-ok-return" class="list-anchor">#</a>

Whether the test is considered running fine. Non-ok tests fail the test run with non-zero exit code.

## testCase.outcome() {#test-case-outcome}
- returns:<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="test-case-outcome-return"/> &#60;"skipped"|"expected"|"unexpected"|"flaky"&#62;<a href="#test-case-outcome-return" class="list-anchor">#</a>

Testing outcome for this test. Note that outcome is not the same as [testResult.status](./api/class-testresult.mdx#test-result-status):
* Test that is expected to fail and actually fails is `'expected'`.
* Test that passes on a second retry is `'flaky'`.

## testCase.titlePath() {#test-case-title-path}
- returns:<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="test-case-title-path-return"/> &#60;[Array]<[string]>&#62;<a href="#test-case-title-path-return" class="list-anchor">#</a>

Returns a list of titles from the root down to this test.

## testCase.annotations {#test-case-annotations}
- type: &#60;[Array]<[Object]>&#62;
  - `type` &#60;[string]&#62; Annotation type, for example `'skip'` or `'fail'`.
  - `description` &#60;[void]|[string]&#62; Optional description.

The list of annotations applicable to the current test. Includes annotations from the test, annotations from all [test.describe(title, callback)](./api/class-test.mdx#test-describe) groups the test belongs to and file-level annotations for the test file.

Annotations are available during test execution through [testInfo.annotations](./api/class-testinfo.mdx#test-info-annotations).

Learn more about [test annotations](./test-annotations.mdx).

## testCase.expectedStatus {#test-case-expected-status}
- type: &#60;"passed"|"failed"|"timedOut"|"skipped"&#62;

Expected test status.
* Tests marked as [test.skip(title, testFunction)](./api/class-test.mdx#test-skip-1) or [test.fixme([condition, description])](./api/class-test.mdx#test-fixme) are expected to be `'skipped'`.
* Tests marked as [test.fail([condition, description])](./api/class-test.mdx#test-fail) are expected to be `'failed'`.
* Other tests are expected to be `'passed'`.

See also [testResult.status](./api/class-testresult.mdx#test-result-status) for the actual status.

## testCase.location {#test-case-location}
- type: &#60;[void]|[Location]&#62;

Location in the source where the test is defined.

## testCase.results {#test-case-results}
- type: &#60;[Array]<[TestResult]>&#62;

Results for each run of this test.

## testCase.retries {#test-case-retries}
- type: &#60;[number]&#62;

The maximum number of retries given to this test in the configuration.

Learn more about [test retries](./test-retries.mdx#retries).

## testCase.suite {#test-case-suite}
- type: &#60;[Suite]&#62;

Suite this test case belongs to.

## testCase.timeout {#test-case-timeout}
- type: &#60;[number]&#62;

The timeout given to the test. Affected by [testConfig.timeout](./api/class-testconfig.mdx#test-config-timeout), [testProject.timeout](./api/class-testproject.mdx#test-project-timeout), [test.setTimeout(timeout)](./api/class-test.mdx#test-set-timeout), [test.slow([condition, description])](./api/class-test.mdx#test-slow) and [testInfo.setTimeout(timeout)](./api/class-testinfo.mdx#test-info-set-timeout).

## testCase.title {#test-case-title}
- type: &#60;[string]&#62;

Test title as passed to the [test(title, testFunction)](./api/class-test.mdx#test-call) call.
