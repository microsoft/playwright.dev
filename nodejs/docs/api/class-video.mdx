When browser context is created with the `recordVideo` option, each page has a video object associated with it.

```js
console.log(await page.video().path());
```


- [video.delete()](/api/class-video.mdx#video-delete)
- [video.path()](/api/class-video.mdx#video-path)
- [video.saveAs(path)](/api/class-video.mdx#video-save-as)

## video.delete() {#video-delete}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.11</font>

- returns:<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="video-delete-return"/> &#60;[Promise]<[void]>&#62;<a href="#video-delete-return" class="list-anchor">#</a>

Deletes the video file. Will wait for the video to finish if necessary.

## video.path() {#video-path}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.8</font>

- returns:<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="video-path-return"/> &#60;[Promise]<[string]>&#62;<a href="#video-path-return" class="list-anchor">#</a>

Returns the file system path this video will be recorded to. The video is guaranteed to be written to the filesystem upon closing the browser context. This method throws when connected remotely.

## video.saveAs(path) {#video-save-as}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.11</font>

- `path`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="video-save-as-option-path"/> &#60;[string]&#62; Path where the video should be saved.<a href="#video-save-as-option-path" class="list-anchor">#</a>
- returns:<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="video-save-as-return"/> &#60;[Promise]<[void]>&#62;<a href="#video-save-as-return" class="list-anchor">#</a>

Saves the video to a user-specified path. It is safe to call this method while the video is still in progress, or after the page has closed. This method waits until the page is closed and the video is fully saved.
