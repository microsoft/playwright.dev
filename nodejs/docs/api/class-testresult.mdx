## testResult.attachments {#test-result-attachments}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[Array]<[Object]>&#62;
  - `name` &#60;[string]&#62; Attachment name.
  - `contentType` &#60;[string]&#62; Content type of this attachment to properly present in the report, for example `'application/json'` or `'image/png'`.
  - `path?` &#60;[string]&#62; Optional path on the filesystem to the attached file.
  - `body?` &#60;[Buffer]&#62; Optional attachment body used instead of a file.

The list of files or buffers attached during the test execution through [testInfo.attachments](/api/class-testinfo.mdx#test-info-attachments).

## testResult.duration {#test-result-duration}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[number]&#62;

Running time in milliseconds.

## testResult.error {#test-result-error}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[TestError]&#62;

First error thrown during test execution, if any. This is equal to the first element in [testResult.errors](/api/class-testresult.mdx#test-result-errors).

## testResult.errors {#test-result-errors}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[Array]<[TestError]>&#62;

Errors thrown during the test execution.

## testResult.retry {#test-result-retry}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[number]&#62;

When test is retries multiple times, each retry attempt is given a sequential number.

Learn more about [test retries](../test-retries.mdx#retries).

## testResult.startTime {#test-result-start-time}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[Date]&#62;

Start time of this particular test run.

## testResult.status {#test-result-status}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;"passed"|"failed"|"timedOut"|"skipped"|"interrupted"&#62;

The status of this test result. See also [testCase.expectedStatus](/api/class-testcase.mdx#test-case-expected-status).

## testResult.stderr {#test-result-stderr}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[Array]<[string]|[Buffer]>&#62;

Anything written to the standard error during the test run.

## testResult.stdout {#test-result-stdout}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[Array]<[string]|[Buffer]>&#62;

Anything written to the standard output during the test run.

## testResult.steps {#test-result-steps}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[Array]<[TestStep]>&#62;

List of steps inside this test run.

## testResult.workerIndex {#test-result-worker-index}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.10</font>

- type: &#60;[number]&#62;

Index of the worker where the test was run. If the test was not run a single time, for example when the user interrupted testing, the only result will have a `workerIndex` equal to `-1`.

Learn more about [parallelism and sharding](../test-parallel.mdx) with Playwright Test.
