[FileChooser] objects are dispatched by the page in the [page.on('filechooser')](/api/class-page.mdx#page-event-file-chooser) event.

```js
// Note that Promise.all prevents a race condition
// between clicking and waiting for the file chooser.
const [fileChooser] = await Promise.all([
  // It is important to call waitForEvent before click to set up waiting.
  page.waitForEvent('filechooser'),
  // Opens the file chooser.
  page.getByText('Upload').click(),
]);
await fileChooser.setFiles('myfile.pdf');
```

## fileChooser.setFiles(files[, options]) {#file-chooser-set-files}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.8</font>

- `files`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="file-chooser-set-files-option-files"/> &#60;[string]|[Array]<[string]>|[Object]|[Array]<[Object]>&#62;<a href="#file-chooser-set-files-option-files" class="list-anchor">#</a>
  - `name` &#60;[string]&#62; File name
  - `mimeType` &#60;[string]&#62; File type
  - `buffer` &#60;[Buffer]&#62; File content
- `options?` &#60;[Object]&#62;
  - `noWaitAfter?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="file-chooser-set-files-option-no-wait-after"/> &#60;[boolean]&#62; Actions that initiate navigations are waiting for these navigations to happen and for pages to start loading. You can opt out of waiting via setting this flag. You would only need this option in the exceptional cases such as navigating to inaccessible pages. Defaults to `false`.<a href="#file-chooser-set-files-option-no-wait-after" class="list-anchor">#</a>
  - `timeout?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="file-chooser-set-files-option-timeout"/> &#60;[number]&#62; Maximum time in milliseconds, defaults to 30 seconds, pass `0` to disable timeout. The default value can be changed by using the [browserContext.setDefaultTimeout(timeout)](/api/class-browsercontext.mdx#browser-context-set-default-timeout) or [page.setDefaultTimeout(timeout)](/api/class-page.mdx#page-set-default-timeout) methods.<a href="#file-chooser-set-files-option-timeout" class="list-anchor">#</a>
- returns:<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="file-chooser-set-files-return"/> &#60;[Promise]<[void]>&#62;<a href="#file-chooser-set-files-return" class="list-anchor">#</a>

Sets the value of the file input this chooser is associated with. If some of the `filePaths` are relative paths, then they are resolved relative to the current working directory. For empty array, clears the selected files.
