## androidDevice.launchBrowser([options]) {#android-device-launch-browser}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.9</font>

- `options?` &#60;[Object]&#62;
  - `acceptDownloads?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="android-device-launch-browser-option-accept-downloads"/> &#60;[boolean]&#62; Whether to automatically download all the attachments. Defaults to `true` where all the downloads are accepted.<a href="#android-device-launch-browser-option-accept-downloads" class="list-anchor">#</a>
  - `baseURL?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="android-device-launch-browser-option-base-url"/> &#60;[string]&#62; When using [page.goto(url[, options])](/api/class-page.mdx#page-goto), [page.route(url, handler[, options])](/api/class-page.mdx#page-route), [page.waitForURL(url[, options])](/api/class-page.mdx#page-wait-for-url), [page.waitForRequest(urlOrPredicate[, options])](/api/class-page.mdx#page-wait-for-request), or [page.waitForResponse(urlOrPredicate[, options])](/api/class-page.mdx#page-wait-for-response) it takes the base URL in consideration by using the [`URL()`](https://developer.mozilla.org/en-US/docs/Web/API/URL/URL) constructor for building the corresponding URL. Examples:<a href="#android-device-launch-browser-option-base-url" class="list-anchor">#</a>
    * baseURL: `http://localhost:3000` and navigating to `/bar.html` results in `http://localhost:3000/bar.html`
    * baseURL: `http://localhost:3000/foo/` and navigating to `./bar.html` results in `http://localhost:3000/foo/bar.html`
    * baseURL: `http://localhost:3000/foo` (without trailing slash) and navigating to `./bar.html` results in `http://localhost:3000/bar.html`
  - `bypassCSP?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="android-device-launch-browser-option-bypass-csp"/> &#60;[boolean]&#62; Toggles bypassing page's Content-Security-Policy.<a href="#android-device-launch-browser-option-bypass-csp" class="list-anchor">#</a>
  - `colorScheme?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="android-device-launch-browser-option-color-scheme"/> &#60;[null]|"light"|"dark"|"no-preference"&#62; Emulates `'prefers-colors-scheme'` media feature, supported values are `'light'`, `'dark'`, `'no-preference'`. See [page.emulateMedia([options])](/api/class-page.mdx#page-emulate-media) for more details. Passing `null` resets emulation to system defaults. Defaults to `'light'`.<a href="#android-device-launch-browser-option-color-scheme" class="list-anchor">#</a>
  - `command?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="android-device-launch-browser-option-command"/> &#60;[string]&#62; Optional package name to launch instead of default Chrome for Android.<a href="#android-device-launch-browser-option-command" class="list-anchor">#</a>
  - `deviceScaleFactor?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="android-device-launch-browser-option-device-scale-factor"/> &#60;[number]&#62; Specify device scale factor (can be thought of as dpr). Defaults to `1`.<a href="#android-device-launch-browser-option-device-scale-factor" class="list-anchor">#</a>
  - `extraHTTPHeaders?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="android-device-launch-browser-option-extra-http-headers"/> &#60;[Object]<[string], [string]>&#62; An object containing additional HTTP headers to be sent with every request.<a href="#android-device-launch-browser-option-extra-http-headers" class="list-anchor">#</a>
  - `forcedColors?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="android-device-launch-browser-option-forced-colors"/> &#60;[null]|"active"|"none"&#62; Emulates `'forced-colors'` media feature, supported values are `'active'`, `'none'`. See [page.emulateMedia([options])](/api/class-page.mdx#page-emulate-media) for more details. Passing `null` resets emulation to system defaults. Defaults to `'none'`.<a href="#android-device-launch-browser-option-forced-colors" class="list-anchor">#</a>
  - `geolocation?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="android-device-launch-browser-option-geolocation"/> &#60;[Object]&#62;<a href="#android-device-launch-browser-option-geolocation" class="list-anchor">#</a>
    - `latitude` &#60;[number]&#62; Latitude between -90 and 90.
    - `longitude` &#60;[number]&#62; Longitude between -180 and 180.
    - `accuracy?` &#60;[number]&#62; Non-negative accuracy value. Defaults to `0`.
  - `hasTouch?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="android-device-launch-browser-option-has-touch"/> &#60;[boolean]&#62; Specifies if viewport supports touch events. Defaults to false.<a href="#android-device-launch-browser-option-has-touch" class="list-anchor">#</a>
  - `httpCredentials?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="android-device-launch-browser-option-http-credentials"/> &#60;[Object]&#62; Credentials for [HTTP authentication](https://developer.mozilla.org/en-US/docs/Web/HTTP/Authentication).<a href="#android-device-launch-browser-option-http-credentials" class="list-anchor">#</a>
    - `username` &#60;[string]&#62;
    - `password` &#60;[string]&#62;
  - `ignoreHTTPSErrors?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="android-device-launch-browser-option-ignore-https-errors"/> &#60;[boolean]&#62; Whether to ignore HTTPS errors when sending network requests.   - `recordHar?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="android-device-launch-browser-option-record-har"/> &#60;[Object]&#62; Enables [HAR](http://www.softwareishard.com/blog/har-12-spec) recording for all pages into `recordHar.path` file. If not specified, the HAR is not recorded. Make sure to await [browserContext.close()](/api/class-browsercontext.mdx#browser-context-close) for the HAR to be saved.<a href="#android-device-launch-browser-option-record-har" class="list-anchor">#</a>
    - `omitContent?` &#60;[boolean]&#62; Optional setting to control whether to omit request content from the HAR. Defaults to `false`. Deprecated, use `content` policy instead.
    - `content?` &#60;"omit"|"embed"|"attach"&#62; Optional setting to control resource content management. If `omit` is specified, content is not persisted. If `attach` is specified, resources are persisted as separate files or entries in the ZIP archive. If `embed` is specified, content is stored inline the HAR file as per HAR specification. Defaults to `attach` for `.zip` output files and to `embed` for all other file extensions.
    - `path` &#60;[string]&#62; Path on the filesystem to write the HAR file to. If the file name ends with `.zip`, `content: 'attach'` is used by default.
    - `mode?` &#60;"full"|"minimal"&#62; When set to `minimal`, only record information necessary for routing from HAR. This omits sizes, timing, page, cookies, security and other types of HAR information that are not used when replaying from HAR. Defaults to `full`.
    - `urlFilter?` &#60;[string]|[RegExp]&#62; A glob or regex pattern to filter requests that are stored in the HAR.     - `dir` &#60;[string]&#62; Path to the directory to put videos into.
    - `size?` &#60;[Object]&#62; Optional dimensions of the recorded videos. If not specified the size will be equal to `viewport` scaled down to fit into 800x800. If `viewport` is not configured explicitly the video size defaults to 800x450. Actual picture of each page will be scaled down if necessary to fit the specified size.
      - `width` &#60;[number]&#62; Video frame width.
      - `height` &#60;[number]&#62; Video frame height.
  - `reducedMotion?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="android-device-launch-browser-option-reduced-motion"/> &#60;[null]|"reduce"|"no-preference"&#62; Emulates `'prefers-reduced-motion'` media feature, supported values are `'reduce'`, `'no-preference'`.   - `screen?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="android-device-launch-browser-option-screen"/> &#60;[Object]&#62; Emulates consistent window screen size available inside web page via `window.screen`.     - `height` &#60;[number]&#62; page height in pixels.
  - `serviceWorkers?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="android-device-launch-browser-option-service-workers"/> &#60;"allow"|"block"&#62; Whether to allow sites to register Service workers. Defaults to `'allow'`.<a href="#android-device-launch-browser-option-service-workers" class="list-anchor">#</a>

    * `'block'`: Playwright will block all registration of Service Workers.
  - `strictSelectors?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="android-device-launch-browser-option-strict-selectors"/> &#60;[boolean]&#62; If specified, enables strict selectors mode for this context.   - `timezoneId?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="android-device-launch-browser-option-timezone-id"/> &#60;[string]&#62; Changes the timezone of the context.   - `userAgent?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="android-device-launch-browser-option-user-agent"/> &#60;[string]&#62; Specific user agent to use in this context.<a href="#android-device-launch-browser-option-user-agent" class="list-anchor">#</a>
  - `videoSize?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="android-device-launch-browser-option-video-size"/> &#60;[Object]&#62; **DEPRECATED** Use `recordVideo` instead.<a href="#android-device-launch-browser-option-video-size" class="list-anchor">#</a>
