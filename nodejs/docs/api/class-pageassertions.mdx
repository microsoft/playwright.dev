The [PageAssertions] class provides assertion methods that can be used to make assertions about the [Page] state in the tests. A new instance of [PageAssertions] is created by calling [expect(page)](./api/class-playwrightassertions.mdx#playwright-assertions-expect-page):

```js
import { test, expect } from '@playwright/test';

test('navigates to login', async ({ page }) => {
  // ...
  await page.click('#login');
  await expect(page).toHaveURL(/.*\/login/);
});
```

- [expect(page).toHaveTitle(titleOrRegExp[, options])](./api/class-pageassertions.mdx#page-assertions-to-have-title)

## expect(page).toHaveScreenshot([options]) {#page-assertions-to-have-screenshot}
- `options` &#60;[Object]&#62;
  - `clip`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-option-clip"/> &#60;[Object]&#62; An object which specifies clipping of the resulting image. Should have the following fields:<a href="#page-assertions-to-have-screenshot-option-clip" class="list-anchor">#</a>
    - `x` &#60;[number]&#62; x-coordinate of top-left corner of clip area
    - `y` &#60;[number]&#62; y-coordinate of top-left corner of clip area
    - `width` &#60;[number]&#62; width of clipping area
    - `height` &#60;[number]&#62; height of clipping area
  - `disableAnimations`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-option-disable-animations"/> &#60;[boolean]&#62; When true, stops CSS animations, CSS transitions and Web Animations. Animations get different treatment depending on their duration:<a href="#page-assertions-to-have-screenshot-option-disable-animations" class="list-anchor">#</a>
  - `fullPage`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-option-full-page"/> &#60;[boolean]&#62; When true, takes a screenshot of the full scrollable page, instead of the currently visible viewport. Defaults to `false`.<a href="#page-assertions-to-have-screenshot-option-full-page" class="list-anchor">#</a>
  - `mask`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-option-mask"/> &#60;[Array]<[Locator]>&#62; Specify locators that should be masked when the screenshot is taken. Masked elements will be overlayed with a pink box `#FF00FF` that completely covers its bounding box.<a href="#page-assertions-to-have-screenshot-option-mask" class="list-anchor">#</a>
  - `maxDiffPixelRatio`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-option-max-diff-pixel-ratio"/> &#60;[number]&#62; An acceptable ratio of pixels that are different to the total amount of pixels, between `0` and `1`, unset by default.<a href="#page-assertions-to-have-screenshot-option-max-diff-pixel-ratio" class="list-anchor">#</a>
  - `maxDiffPixels`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-option-max-diff-pixels"/> &#60;[number]&#62; An acceptable amount of pixels that could be different, unset by default.<a href="#page-assertions-to-have-screenshot-option-max-diff-pixels" class="list-anchor">#</a>
  - `omitBackground`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-option-omit-background"/> &#60;[boolean]&#62; Hides default white background and allows capturing screenshots with transparency. Not applicable to `jpeg` images. Defaults to `false`.<a href="#page-assertions-to-have-screenshot-option-omit-background" class="list-anchor">#</a>
  - `threshold`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-option-threshold"/> &#60;[number]&#62; An acceptable percieved color difference in the [YIQ color space](https://en.wikipedia.org/wiki/YIQ) between the same pixel in compared images, between zero (strict) and one (lax), default is configurable with `TestConfig.expect`. Defaults to `0.2`.<a href="#page-assertions-to-have-screenshot-option-threshold" class="list-anchor">#</a>
  - `timeout`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-option-timeout"/> &#60;[number]&#62; Time to retry the assertion for. Defaults to `timeout` in `TestConfig.expect`.<a href="#page-assertions-to-have-screenshot-option-timeout" class="list-anchor">#</a>
- returns:<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="page-assertions-to-have-screenshot-return"/> &#60;[Promise]<[void]>&#62;<a href="#page-assertions-to-have-screenshot-return" class="list-anchor">#</a>

Ensures that the page resolves to a given screenshot. This function will re-take screenshots until it matches with the saved expectation.

If there's no expectation yet, it will wait until two consecutive screenshots yield the same result, and save the last one as an expectation.

```js
await expect(page).toHaveScreenshot();
```

## expect(page).toHaveTitle(titleOrRegExp[, options]) {#page-assertions-to-have-title}
- `titleOrRegExp`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="page-assertions-to-have-title-option-title-or-reg-exp"/> &#60;[string]|[RegExp]&#62; Expected title or RegExp.<a href="#page-assertions-to-have-title-option-title-or-reg-exp" class="list-anchor">#</a>
- `options` &#60;[Object]&#62;
  - `timeout`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="page-assertions-to-have-title-option-timeout"/> &#60;[number]&#62; Time to retry the assertion for. Defaults to `timeout` in `TestConfig.expect`.<a href="#page-assertions-to-have-title-option-timeout" class="list-anchor">#</a>
- returns:<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="page-assertions-to-have-title-return"/> &#60;[Promise]<[void]>&#62;<a href="#page-assertions-to-have-title-return" class="list-anchor">#</a>

Ensures the page has the given title.

```js
await expect(page).toHaveTitle(/.*checkout/);
```

## expect(page).toHaveURL(urlOrRegExp[, options]) {#page-assertions-to-have-url}
- `urlOrRegExp`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="page-assertions-to-have-url-option-url-or-reg-exp"/> &#60;[string]|[RegExp]&#62; Expected substring or RegExp.<a href="#page-assertions-to-have-url-option-url-or-reg-exp" class="list-anchor">#</a>
- `options` &#60;[Object]&#62;
  - `timeout`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="page-assertions-to-have-url-option-timeout"/> &#60;[number]&#62; Time to retry the assertion for. Defaults to `timeout` in `TestConfig.expect`.<a href="#page-assertions-to-have-url-option-timeout" class="list-anchor">#</a>
- returns:<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="page-assertions-to-have-url-return"/> &#60;[Promise]<[void]>&#62;<a href="#page-assertions-to-have-url-return" class="list-anchor">#</a>

Ensures the page is navigated to the given URL.

```js
await expect(page).toHaveURL(/.*checkout/);
```

## expect(page).not {#page-assertions-not}
- type: &#60;[PageAssertions]&#62;

Makes the assertion check for the opposite condition. For example, this code tests that the page URL doesn't contain `"error"`:

```js
await expect(page).not.toHaveURL('error');
```
