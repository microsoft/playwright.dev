This API is used for Web API testing. You can use it to trigger API endpoints, configure micro-services, prepare environment or the service to your e2e test. When used on [Page] or a [BrowserContext], this API will automatically use the cookies from the corresponding [BrowserContext]. This means that if you log in using this API, your e2e test will be logged in and vice versa.


- [fetchRequest.dispose()](./api/class-fetchrequest.mdx#fetch-request-dispose)
- [fetchRequest.fetch(urlOrRequest[, options])](./api/class-fetchrequest.mdx#fetch-request-fetch)
- [fetchRequest.get(urlOrRequest[, options])](./api/class-fetchrequest.mdx#fetch-request-get)
- [fetchRequest.post(urlOrRequest[, options])](./api/class-fetchrequest.mdx#fetch-request-post)

## fetchRequest.dispose() {#fetch-request-dispose}
- returns:<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="fetch-request-dispose-return"/> &#60;[Promise]<[void]>&#62;<a href="#fetch-request-dispose-return" class="list-anchor">#</a>

All responses received through [fetchRequest.fetch(urlOrRequest[, options])](./api/class-fetchrequest.mdx#fetch-request-fetch), [fetchRequest.get(urlOrRequest[, options])](./api/class-fetchrequest.mdx#fetch-request-get), [fetchRequest.post(urlOrRequest[, options])](./api/class-fetchrequest.mdx#fetch-request-post) and other methods are stored in the memory, so that you can later call [fetchResponse.body()](./api/class-fetchresponse.mdx#fetch-response-body). This method discards all stored responses, and makes [fetchResponse.body()](./api/class-fetchresponse.mdx#fetch-response-body) throw "Response disposed" error.

## fetchRequest.fetch(urlOrRequest[, options]) {#fetch-request-fetch}
- `urlOrRequest`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="fetch-request-fetch-option-url-or-request"/> &#60;[string]|[Request]&#62; Target URL or Request to get all fetch parameters from.<a href="#fetch-request-fetch-option-url-or-request" class="list-anchor">#</a>
- `options` &#60;[Object]&#62;
  - `data`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="fetch-request-fetch-option-data"/> &#60;[string]|[Buffer]&#62; Allows to set post data of the fetch.<a href="#fetch-request-fetch-option-data" class="list-anchor">#</a>
  - `failOnStatusCode`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="fetch-request-fetch-option-fail-on-status-code"/> &#60;[boolean]&#62; Whether to throw on response codes other than 2xx and 3xx. By default response object is returned for all status codes.<a href="#fetch-request-fetch-option-fail-on-status-code" class="list-anchor">#</a>
  - `headers`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="fetch-request-fetch-option-headers"/> &#60;[Object]<[string], [string]>&#62; Allows to set HTTP headers.<a href="#fetch-request-fetch-option-headers" class="list-anchor">#</a>
  - `method`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="fetch-request-fetch-option-method"/> &#60;[string]&#62; If set changes the fetch method (e.g. PUT or POST). If not specified, GET method is used.<a href="#fetch-request-fetch-option-method" class="list-anchor">#</a>
  - `params`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="fetch-request-fetch-option-params"/> &#60;[Object]<[string], [string]>&#62; Query parameters to be send with the URL.<a href="#fetch-request-fetch-option-params" class="list-anchor">#</a>
  - `timeout`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="fetch-request-fetch-option-timeout"/> &#60;[number]&#62; Request timeout in milliseconds.<a href="#fetch-request-fetch-option-timeout" class="list-anchor">#</a>
- returns:<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="fetch-request-fetch-return"/> &#60;[Promise]<[FetchResponse]>&#62;<a href="#fetch-request-fetch-return" class="list-anchor">#</a>

Sends HTTP(S) fetch and returns its response. The method will populate fetch cookies from the context and update context cookies from the response. The method will automatically follow redirects.
