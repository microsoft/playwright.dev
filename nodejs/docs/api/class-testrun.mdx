Controls which tests will run and their expected status. A [TestRun] is available during [reporter.preprocess()](/api/class-reporter.mdx#reporter-preprocess). Setup and teardown projects cannot be changed.


---

## Methods

### exclude {/* #test-run-exclude */}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.62</font><x-search>testRun.exclude</x-search>

Excludes a test or suite from the run. Excluded tests do not appear in the report and their bodies are not executed.

**Usage**

```js
testRun.exclude(test);
```

**Arguments**
- `test` [TestCase] | [Suite]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="test-run-exclude-option-test"/><a href="#test-run-exclude-option-test" class="list-anchor">#</a>
  
  Test or suite to exclude. The root suite cannot be excluded.

---

### fail {/* #test-run-fail */}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.62</font><x-search>testRun.fail</x-search>

Marks a test or every test in a suite as "should fail". Playwright runs the tests and ensures they are actually failing, useful for documenting broken functionality until it is fixed.

**Usage**

```js
testRun.fail(test);
testRun.fail(test, reason);
```

**Arguments**
- `test` [TestCase] | [Suite]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="test-run-fail-option-test"/><a href="#test-run-fail-option-test" class="list-anchor">#</a>
  
  Test or suite to mark as expected-to-fail.
- `reason` [string] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="test-run-fail-option-reason"/><a href="#test-run-fail-option-reason" class="list-anchor">#</a>
  
  Optional explanation surfaced as the annotation description.

---

### fixme {/* #test-run-fixme */}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.62</font><x-search>testRun.fixme</x-search>

Marks a test or every test in a suite as fixme. The test bodies are not executed and the tests are reported as skipped, with the intention to fix them.

**Usage**

```js
testRun.fixme(test);
testRun.fixme(test, reason);
```

**Arguments**
- `test` [TestCase] | [Suite]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="test-run-fixme-option-test"/><a href="#test-run-fixme-option-test" class="list-anchor">#</a>
  
  Test or suite to mark as fixme.
- `reason` [string] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="test-run-fixme-option-reason"/><a href="#test-run-fixme-option-reason" class="list-anchor">#</a>
  
  Optional explanation surfaced as the annotation description.

---

### skip {/* #test-run-skip */}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.62</font><x-search>testRun.skip</x-search>

Skips a test or every test in a suite. The test bodies are not executed and the tests are reported as skipped.

**Usage**

```js
testRun.skip(test);
testRun.skip(test, reason);
```

**Arguments**
- `test` [TestCase] | [Suite]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="test-run-skip-option-test"/><a href="#test-run-skip-option-test" class="list-anchor">#</a>
  
  Test or suite to skip.
- `reason` [string] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="test-run-skip-option-reason"/><a href="#test-run-skip-option-reason" class="list-anchor">#</a>
  
  Optional explanation surfaced as the annotation description.

---

### skipSharding {/* #test-run-skip-sharding */}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.62</font><x-search>testRun.skipSharding</x-search>

Disables Playwright's built-in shard filter for this run, leaving sharding to the reporter. Reporters typically implement their own sharding by calling [testRun.exclude()](/api/class-testrun.mdx#test-run-exclude) on out-of-shard tests.

**Usage**

```js
testRun.skipSharding();
```
