Playwright Test provides a global `store` object for passing values between project setup and tests. It is an error to call store methods outside of setup and tests.

<Tabs
  groupId="js-flavor"
  defaultValue="ts"
  values={[
    {label: 'TypeScript', value: 'ts'},
    {label: 'JavaScript', value: 'js'}
  ]
}>
<TabItem value="ts">

```js
import { setup, store } from '@playwright/test';

setup('sign in', async ({ page, context }) => {
  // Save signed-in state to an entry named 'github-test-user'.
  const contextState = await context.storageState();
  await store.set('test-user', contextState)
});
```

</TabItem>
<TabItem value="js">

```js
const { setup, store } = require('@playwright/test');

setup('sign in', async ({ page, context }) => {
  // Save signed-in state to an entry named 'github-test-user'.
  const contextState = await context.storageState();
  await store.set('test-user', contextState)
});
```

</TabItem>
</Tabs>

## Methods

### get {#test-store-get}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.29</font>


Get named item from the store. Returns undefined if there is no value with given name.

**Usage**

```js
await testStore.get(name);
```

**Arguments**
- `name` [string]<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="test-store-get-option-name"/><a href="#test-store-get-option-name" class="list-anchor">#</a>
  
  Item name.

### set {#test-store-set}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.29</font>


Set value to the store.

**Usage**

```js
await testStore.set(name, value);
```

**Arguments**
- `name` [string]<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="test-store-set-option-name"/><a href="#test-store-set-option-name" class="list-anchor">#</a>
  
  Item name.
- `value` [Object]<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="test-store-set-option-value"/><a href="#test-store-set-option-value" class="list-anchor">#</a>
  
  Item value. The value must be serializable to JSON. Passing `undefined` deletes the entry with given name.
