## webSocket.waitForEvent(event[, optionsOrPredicate, options]) {#web-socket-wait-for-event}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.8</font>

- `event`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="web-socket-wait-for-event-option-event"/> &#60;[string]&#62; Event name, same one would pass into `webSocket.on(event)`.<a href="#web-socket-wait-for-event-option-event" class="list-anchor">#</a>
- `optionsOrPredicate?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="web-socket-wait-for-event-option-options-or-predicate"/> &#60;[function]|[Object]&#62; Either a predicate that receives an event or an options object. Optional.<a href="#web-socket-wait-for-event-option-options-or-predicate" class="list-anchor">#</a>
  - `predicate` &#60;[function]&#62; receives the event data and resolves to truthy value when the waiting should resolve.
  - `timeout?` &#60;[number]&#62; maximum time to wait for in milliseconds. Defaults to `30000` (30 seconds). Pass `0` to disable timeout. The default value can be changed by using the [browserContext.setDefaultTimeout(timeout)](/api/class-browsercontext.mdx#browser-context-set-default-timeout).
- `options?` &#60;[Object]&#62;
  - `predicate?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="web-socket-wait-for-event-option-predicate"/> &#60;[function]&#62; Receives the event data and resolves to truthy value when the waiting should resolve.<a href="#web-socket-wait-for-event-option-predicate" class="list-anchor">#</a>
- returns:<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="web-socket-wait-for-event-return"/> &#60;[Promise]<[Object]>&#62;<a href="#web-socket-wait-for-event-return" class="list-anchor">#</a>

Waits for event to fire and passes its value into the predicate function. Returns when the predicate returns truthy value. Will throw an error if the webSocket is closed before the event is fired. Returns the event data value.
