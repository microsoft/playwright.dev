## Events

### on('close') {#browser-server-event-close}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.8</font>


Emitted when the browser server closes.

**Usage**

```js
browserServer.on('close', data => {});
```

---

## Methods

### close {#browser-server-close}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.8</font>


Closes the browser gracefully and makes sure the process is terminated.

**Usage**

```js
await browserServer.close();
```

---

### kill {#browser-server-kill}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.8</font>


Kills the browser process and waits for the process to exit.

**Usage**

```js
await browserServer.kill();
```

---

### process {#browser-server-process}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.8</font>


Spawned browser application process.

**Usage**

```js
browserServer.process();
```

**Returns**
- [ChildProcess]<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="browser-server-process-return"/><a href="#browser-server-process-return" class="list-anchor">#</a>

---

### wsEndpoint {#browser-server-ws-endpoint}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.8</font>


Browser websocket url.

Browser websocket endpoint which can be used as an argument to [browserType.connect()](/api/class-browsertype.mdx#browser-type-connect) to establish connection to the browser.

**Usage**

```js
browserServer.wsEndpoint();
```
