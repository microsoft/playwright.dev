## response.fromServiceWorker() {#response-from-service-worker}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.23</font>

Indicates whether this Response was fulfilled by a Service Worker's Fetch Handler (i.e. via [FetchEvent.respondWith](https://developer.mozilla.org/en-US/docs/Web/API/FetchEvent/respondWith)).

## response.headerValue(name) {#response-header-value}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.15</font>

- `name`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="response-header-value-option-name"/> &#60;[string]&#62; Name of the header.<a href="#response-header-value-option-name" class="list-anchor">#</a>
- returns:<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="response-header-value-return"/> &#60;[Promise]<[null]|[string]>&#62;<a href="#response-header-value-return" class="list-anchor">#</a>

Returns the value of the header matching the name. The name is case insensitive. If multiple headers have the same name (except `set-cookie`), they are returned as a list separated by `, `. For `set-cookie`, the `\n` separator is used. If no headers are found, `null` is returned.

## response.headerValues(name) {#response-header-values}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.15</font>

- `name`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="response-header-values-option-name"/> &#60;[string]&#62; Name of the header.<a href="#response-header-values-option-name" class="list-anchor">#</a>
- returns:<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="response-header-values-return"/> &#60;[Promise]<[Array]<[string]>>&#62;<a href="#response-header-values-return" class="list-anchor">#</a>

Returns all values of the headers matching the name, for example `set-cookie`. The name is case insensitive.

## response.headers() {#response-headers}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.8</font>

- returns:<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="response-headers-return"/> &#60;[Object]<[string], [string]>&#62;<a href="#response-headers-return" class="list-anchor">#</a>

An object with the response HTTP headers. The header names are lower-cased. Note that this method does not return security-related headers, including cookie-related ones. You can use [response.allHeaders()](/api/class-response.mdx#response-all-headers) for complete list of headers that include `cookie` information.

## response.headersArray() {#response-headers-array}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.15</font>

- returns:<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="response-headers-array-return"/> &#60;[Promise]<[Array]<[Object]>>&#62;<a href="#response-headers-array-return" class="list-anchor">#</a>
  - `name` &#60;[string]&#62; Name of the header.
  - `value` &#60;[string]&#62; Value of the header.

An array with all the request HTTP headers associated with this response. Unlike [response.allHeaders()](/api/class-response.mdx#response-all-headers), header names are NOT lower-cased. Headers with multiple entries, such as `Set-Cookie`, appear in the array multiple times.

## response.json() {#response-json}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.8</font>

Returns the JSON representation of response body.

This method will throw if the response body is not parsable via `JSON.parse`.

## response.securityDetails() {#response-security-details}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.13</font>

- returns:<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="response-security-details-return"/> &#60;[Promise]<[null]|[Object]>&#62;<a href="#response-security-details-return" class="list-anchor">#</a>
  - `issuer?` &#60;[string]&#62; Common Name component of the Issuer field. from the certificate. This should only be used for informational purposes. Optional.
  - `protocol?` &#60;[string]&#62; The specific TLS protocol used. (e.g. `TLS 1.3`). Optional.
  - `subjectName?` &#60;[string]&#62; Common Name component of the Subject field from the certificate. This should only be used for informational purposes. Optional.
  - `validFrom?` &#60;[number]&#62; Unix timestamp (in seconds) specifying when this cert becomes valid. Optional.
  - `validTo?` &#60;[number]&#62; Unix timestamp (in seconds) specifying when this cert becomes invalid. Optional.

Returns SSL and other security information.
