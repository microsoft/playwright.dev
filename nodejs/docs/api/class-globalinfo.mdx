`GlobalInfo` contains information on the overall test run. The information spans projects and tests. Some reporters show global info.

You can write to GlobalInfo via your Global Setup hook, and read from it in a [Custom Reporter](../test-reporters.mdx):

<Tabs
  groupId="js-flavor"
  defaultValue="ts"
  values={[
    {label: 'TypeScript', value: 'ts'},
    {label: 'JavaScript', value: 'js'}
  ]
}>
<TabItem value="ts">

```ts
// global-setup.ts
import { chromium, FullConfig, GlobalInfo } from '@playwright/test';

async function globalSetup(config: FullConfig, info: GlobalInfo) {
  await info.attach('agent.config.txt', { path: './agent.config.txt' });
}

export default globalSetup;
```

</TabItem>
<TabItem value="js">

```js
// global-setup.js
module.exports = async (config, info) => {
  await info.attach('agent.config.txt', { path: './agent.config.txt' });
};
```

</TabItem>
</Tabs>

Access the attachments from the Root Suite in the Reporter:

<Tabs
  groupId="js-flavor"
  defaultValue="ts"
  values={[
    {label: 'TypeScript', value: 'ts'},
    {label: 'JavaScript', value: 'js'}
  ]
}>
<TabItem value="ts">

```ts
// my-awesome-reporter.ts
import { Reporter } from '@playwright/test/reporter';

class MyReporter implements Reporter {
  private _suite;

  onBegin(config, suite) {
    this._suite = suite;
  }

  onEnd(result) {
    console.log(`Finished the run with ${this._suite.attachments.length} global attachments!`);
  }
}
export default MyReporter;
```

</TabItem>
<TabItem value="js">

```js
// my-awesome-reporter.js
// @ts-check

/** @implements {import('@playwright/test/reporter').Reporter} */
class MyReporter {
  onBegin(config, suite) {
    this._suite = suite;
  }

  onEnd(result) {
    console.log(`Finished the run with ${this._suite.attachments.length} global attachments!`);
  }
}

module.exports = MyReporter;
```

</TabItem>
</Tabs>

Finally, specify `globalSetup` in the configuration file and `reporter`:

<Tabs
  groupId="js-flavor"
  defaultValue="ts"
  values={[
    {label: 'TypeScript', value: 'ts'},
    {label: 'JavaScript', value: 'js'}

```ts
// playwright.config.ts
import { PlaywrightTestConfig } from '@playwright/test';

const config: PlaywrightTestConfig = {
  globalSetup: require.resolve('./global-setup'),
  reporter: require.resolve('./my-awesome-reporter'),
};
export default config;
```

</TabItem>
<TabItem value="js">

```js
// playwright.config.js
// @ts-check
/** @type {import('@playwright/test').PlaywrightTestConfig} */
const config = {
  globalSetup: require.resolve('./global-setup'),
  reporter: require.resolve('./my-awesome-reporter'),

module.exports = config;
```

</TabItem>
</Tabs>

## globalInfo.attach(name[, options]) {#global-info-attach}
- `name`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="global-info-attach-option-name"/> &#60;[string]&#62; Attachment name.<a href="#global-info-attach-option-name" class="list-anchor">#</a>
- `options?` &#60;[Object]&#62;
  - `body?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="global-info-attach-option-body"/> &#60;[string]|[Buffer]&#62; Attachment body. Mutually exclusive with `path`.<a href="#global-info-attach-option-body" class="list-anchor">#</a>
  - `contentType?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="global-info-attach-option-content-type"/> &#60;[string]&#62; Optional content type of this attachment to properly present in the report, for example `'application/json'` or `'image/png'`. If omitted, content type is inferred based on the `path`, or defaults to `text/plain` for [string] attachments and `application/octet-stream` for [Buffer] attachments.<a href="#global-info-attach-option-content-type" class="list-anchor">#</a>
  - `path?`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="global-info-attach-option-path"/> &#60;[string]&#62; Path on the filesystem to the attached file. Mutually exclusive with `body`.<a href="#global-info-attach-option-path" class="list-anchor">#</a>
- returns:<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="global-info-attach-return"/> &#60;[Promise]<[void]>&#62;<a href="#global-info-attach-return" class="list-anchor">#</a>

Attach a value or a file from disk to the overall test run. Some reporters show global attachments. Either `path` or `body` must be specified, but not both.

See [testInfo.attach(name[, options])](./api/class-testinfo.mdx#test-info-attach) if you are looking for test-scoped attachments.

:::note
[globalInfo.attach(name[, options])](./api/class-globalinfo.mdx#global-info-attach) automatically takes care of copying attached files to a location that is accessible to reporters. You can safely remove the attachment after awaiting the attach call.
:::

## globalInfo.attachments() {#global-info-attachments}
- returns:<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="global-info-attachments-return"/> &#60;[Array]<[Object]>&#62;<a href="#global-info-attachments-return" class="list-anchor">#</a>
  - `name` &#60;[string]&#62; Attachment name.
  - `contentType` &#60;[string]&#62; Content type of this attachment to properly present in the report, for example `'application/json'` or `'image/png'`.
  - `path?` &#60;[string]&#62; Optional path on the filesystem to the attached file.
  - `body?` &#60;[Buffer]&#62; Optional attachment body used instead of a file.

The list of files or buffers attached to the overall test run. Some reporters show global attachments.

To add an attachment, use [globalInfo.attach(name[, options])](./api/class-globalinfo.mdx#global-info-attach). See [testInfo.attachments](./api/class-testinfo.mdx#test-info-attachments) if you are looking for test-scoped attachments.
