For every attachment downloaded by the page, [Page.onDownload(handler)](./api/class-page.mdx#pageondownloadhandler) event is emitted. If you create a browser context with the `acceptDownloads` set, all these attachments are going to be downloaded into a temporary folder. You can obtain the download url, file system path and payload stream using the [Download] object from the event.

You can specify where to persist downloaded files using the `downloadsPath` option in [BrowserType.launch([options])](./api/class-browsertype.mdx#browsertypelaunchoptions).

:::note
Unless `downloadsPath` is set, downloaded files are deleted when the browser context that produced them is closed.
:::

#### Variations

If you have no idea what initiates the download, you can still handle the event:

```java
page.onDownload(download -> System.out.println(download.path()));
```

Note that handling the event forks the control flow and makes script harder to follow. Your scenario might end while you are downloading a file since your main control flow is not awaiting for this operation to resolve.
