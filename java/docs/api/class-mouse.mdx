The Mouse class operates in main-frame CSS pixels relative to the top-left corner of the viewport.

:::tip

If you want to debug where the mouse moved, you can use the [Trace viewer](../trace-viewer-intro.mdx) or [Playwright Inspector](../running-tests.mdx). A red dot showing the location of the mouse will be shown for every mouse action.
:::

Every `page` object has its own Mouse, accessible with [Page.mouse()](/api/class-page.mdx#page-mouse).

```java
// Using ‘page.mouse’ to trace a 100x100 square.
page.mouse().move(0, 0);
page.mouse().down();
page.mouse().move(0, 100);
page.mouse().move(100, 100);
page.mouse().move(100, 0);
page.mouse().move(0, 0);
page.mouse().up();
```

## Methods

### click {#mouse-click}

<font size="2" style={{position: "relative", top: "-20px"}}>Added before v1.9</font><x-search>mouse.click</x-search>

Shortcut for [Mouse.move()](/api/class-mouse.mdx#mouse-move), [Mouse.down()](/api/class-mouse.mdx#mouse-down), [Mouse.up()](/api/class-mouse.mdx#mouse-up).

**Usage**

```java
Mouse.click(x, y);
Mouse.click(x, y, options);
```

**Arguments**
- `x` [double]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="mouse-click-option-x"/><a href="#mouse-click-option-x" class="list-anchor">#</a>
  
  X coordinate relative to the main frame's viewport in CSS pixels.
- `y` [double]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="mouse-click-option-y"/><a href="#mouse-click-option-y" class="list-anchor">#</a>
  
  Y coordinate relative to the main frame's viewport in CSS pixels.
- `options` `Mouse.ClickOptions` *(optional)*
  - `setButton` `enum MouseButton { LEFT, RIGHT, MIDDLE }` *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="mouse-click-option-button"/><a href="#mouse-click-option-button" class="list-anchor">#</a>
    
    Defaults to `left`.
  - `setClickCount` [int] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="mouse-click-option-click-count"/><a href="#mouse-click-option-click-count" class="list-anchor">#</a>
    
    defaults to 1. See [UIEvent.detail].
  - `setDelay` [double] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="mouse-click-option-delay"/><a href="#mouse-click-option-delay" class="list-anchor">#</a>
    
    Time to wait between `mousedown` and `mouseup` in milliseconds. Defaults to 0.

**Returns**
- [void]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="mouse-click-return"/><a href="#mouse-click-return" class="list-anchor">#</a>

### wheel {#mouse-wheel}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.15</font><x-search>mouse.wheel</x-search>

Dispatches a `wheel` event. This method is usually used to manually scroll the page. See [scrolling](../input.mdx#scrolling) for alternative ways to scroll.

:::note

Wheel events may cause scrolling if they are not handled, and this method does not wait for the scrolling to finish before returning.
:::

**Usage**

```java
Mouse.wheel(deltaX, deltaY);
```

**Arguments**
- `deltaX` [double]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="mouse-wheel-option-delta-x"/><a href="#mouse-wheel-option-delta-x" class="list-anchor">#</a>
  
  Pixels to scroll horizontally.
- `deltaY` [double]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="mouse-wheel-option-delta-y"/><a href="#mouse-wheel-option-delta-y" class="list-anchor">#</a>
  
  Pixels to scroll vertically.

**Returns**
- [void]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="mouse-wheel-return"/><a href="#mouse-wheel-return" class="list-anchor">#</a>
