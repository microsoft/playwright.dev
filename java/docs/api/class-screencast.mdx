### showChapter {#screencast-show-chapter}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.59</font><x-search>screencast.showChapter</x-search>

Shows a chapter overlay with a title and optional description, centered on the page with a blurred backdrop. Useful for narrating video recordings. The overlay is removed after the specified duration, or 2000ms.

**Usage**

```java
Screencast.showChapter(title);
Screencast.showChapter(title, options);
```

**Arguments**
- `title` [String]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="screencast-show-chapter-option-title"/><a href="#screencast-show-chapter-option-title" class="list-anchor">#</a>
  
  Title text displayed prominently in the overlay.
- `options` `Screencast.ShowChapterOptions` *(optional)*
  - `setDescription` [String] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="screencast-show-chapter-option-description"/><a href="#screencast-show-chapter-option-description" class="list-anchor">#</a>
    
    Optional description text displayed below the title.
  - `setDuration` [double] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="screencast-show-chapter-option-duration"/><a href="#screencast-show-chapter-option-duration" class="list-anchor">#</a>
    
    Duration in milliseconds after which the overlay is automatically removed. Defaults to `2000`.

**Returns**
- [void]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="screencast-show-chapter-return"/><a href="#screencast-show-chapter-return" class="list-anchor">#</a>

### showOverlay {#screencast-show-overlay}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.59</font><x-search>screencast.showOverlay</x-search>

Adds an overlay with the given HTML content. The overlay is displayed on top of the page until removed. Returns a disposable that removes the overlay when disposed.

**Usage**

```java
Screencast.showOverlay(html);
Screencast.showOverlay(html, options);
```

**Arguments**
- `html` [String]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="screencast-show-overlay-option-html"/><a href="#screencast-show-overlay-option-html" class="list-anchor">#</a>
  
  HTML content for the overlay.
- `options` `Screencast.ShowOverlayOptions` *(optional)*
  - `setDuration` [double] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="screencast-show-overlay-option-duration"/><a href="#screencast-show-overlay-option-duration" class="list-anchor">#</a>
    
    Duration in milliseconds after which the overlay is automatically removed. Overlay stays until dismissed if not provided.

**Returns**
- [Disposable]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="screencast-show-overlay-return"/><a href="#screencast-show-overlay-return" class="list-anchor">#</a>

### start {#screencast-start}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.59</font><x-search>screencast.start</x-search>

Starts the screencast. When [setPath](/api/class-screencast.mdx#screencast-start-option-path) is provided, it saves video recording to the specified file. When [setOnFrame](/api/class-screencast.mdx#screencast-start-option-on-frame) is provided, delivers JPEG-encoded frames to the callback. Both can be used together.

**Usage**

**Arguments**
- `options` `Screencast.StartOptions` *(optional)*
  - `setOnFrame` [Consumer]&lt;ScreencastFrame&gt; *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="screencast-start-option-on-frame"/><a href="#screencast-start-option-on-frame" class="list-anchor">#</a>
    - `setData` [byte&#91;&#93;]
      
      JPEG-encoded frame data.
    
    Callback that receives JPEG-encoded frame data.
  - `setPath` [Path] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="screencast-start-option-path"/><a href="#screencast-start-option-path" class="list-anchor">#</a>
    
    Path where the video should be saved when the screencast is stopped. When provided, video recording is started.
  - `setQuality` [int] *(optional)*<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="screencast-start-option-quality"/><a href="#screencast-start-option-quality" class="list-anchor">#</a>
    
    The quality of the image, between 0-100.

**Returns**
- [Disposable]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="screencast-start-return"/><a href="#screencast-start-return" class="list-anchor">#</a>

### stop {#screencast-stop}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.59</font><x-search>screencast.stop</x-search>

Stops the screencast and video recording if active. If a video was being recorded, saves it to the path specified in [Screencast.start()](/api/class-screencast.mdx#screencast-start).

**Usage**

```java
Screencast.stop();
```

**Returns**
- [void]<a aria-hidden="true" tabIndex="-1" class="list-anchor-link" id="screencast-stop-return"/><a href="#screencast-stop-return" class="list-anchor">#</a>
