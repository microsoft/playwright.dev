The Touchscreen class operates in main-frame CSS pixels relative to the top-left corner of the viewport. Methods on the touchscreen can only be used in browser contexts that have been initialized with `hasTouch` set to true.


- [Touchscreen.TapAsync(x, y)](./api/class-touchscreen.mdx#touchscreen-tap)

## Touchscreen.TapAsync(x, y) {#touchscreen-tap}
- `x`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="touchscreen-tap-option-x"/> &#60;[double]&#62;<a href="#touchscreen-tap-option-x" class="list-anchor">#</a>
- `y`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="touchscreen-tap-option-y"/> &#60;[double]&#62;<a href="#touchscreen-tap-option-y" class="list-anchor">#</a>
- returns:<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="touchscreen-tap-return"/> &#60;[void]&#62;<a href="#touchscreen-tap-return" class="list-anchor">#</a>

Dispatches a `touchstart` and `touchend` event with a single touch at the position (`x`,`y`).
