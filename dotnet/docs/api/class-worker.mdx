The Worker class represents a [WebWorker](https://developer.mozilla.org/en-US/docs/Web/API/Web_Workers_API). `worker` event is emitted on the page object to signal a worker creation. `close` event is emitted on the worker object when the worker is gone.

```csharp
page.Worker += (_, worker) =>
{
    Console.WriteLine($"Worker created: {worker.Url}");
    worker.Close += (_, _) => Console.WriteLine($"Worker closed {worker.Url}");
};

Console.WriteLine("Current Workers:");
foreach(var pageWorker in page.Workers)
{
    Console.WriteLine($"\tWorker: {pageWorker.Url}");
}
```

## event Worker.Close {#worker-event-close}
- type: &#60;[Worker]&#62;

Emitted when this dedicated [WebWorker](https://developer.mozilla.org/en-US/docs/Web/API/Web_Workers_API) is terminated.

## Worker.EvaluateAsync(expression, arg) {#worker-evaluate}
- `expression`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="worker-evaluate-option-expression"/> &#60;[string]&#62; JavaScript expression to be evaluated in the browser context. If it looks like a function declaration, it is interpreted as a function. Otherwise, evaluated as an expression.<a href="#worker-evaluate-option-expression" class="list-anchor">#</a>
- `arg`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="worker-evaluate-option-arg"/> &#60;[EvaluationArgument]?&#62; Optional argument to pass to `expression`.<a href="#worker-evaluate-option-arg" class="list-anchor">#</a>
- returns:<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="worker-evaluate-return"/> &#60;[object]&#62;<a href="#worker-evaluate-return" class="list-anchor">#</a>

Returns the return value of `expression`.

If the function passed to the [Worker.EvaluateAsync(expression, arg)](./api/class-worker.mdx#worker-evaluate) returns a [Promise], then [Worker.EvaluateAsync(expression, arg)](./api/class-worker.mdx#worker-evaluate) would wait for the promise to resolve and return its value.

If the function passed to the [Worker.EvaluateAsync(expression, arg)](./api/class-worker.mdx#worker-evaluate) returns a non-[Serializable] value, then [Worker.EvaluateAsync(expression, arg)](./api/class-worker.mdx#worker-evaluate) returns `undefined`. Playwright also supports transferring some additional values that are not serializable by `JSON`: `-0`, `NaN`, `Infinity`, `-Infinity`.

## Worker.EvaluateHandleAsync(expression, arg) {#worker-evaluate-handle}
- `expression`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="worker-evaluate-handle-option-expression"/> &#60;[string]&#62; JavaScript expression to be evaluated in the browser context. If it looks like a function declaration, it is interpreted as a function. Otherwise, evaluated as an expression.<a href="#worker-evaluate-handle-option-expression" class="list-anchor">#</a>
- `arg`<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="worker-evaluate-handle-option-arg"/> &#60;[EvaluationArgument]?&#62; Optional argument to pass to `expression`.<a href="#worker-evaluate-handle-option-arg" class="list-anchor">#</a>
- returns:<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="worker-evaluate-handle-return"/> &#60;[JSHandle]&#62;<a href="#worker-evaluate-handle-return" class="list-anchor">#</a>

Returns the return value of `expression` as a [JSHandle].

The only difference between [Worker.EvaluateAsync(expression, arg)](./api/class-worker.mdx#worker-evaluate) and [Worker.EvaluateHandleAsync(expression, arg)](./api/class-worker.mdx#worker-evaluate-handle) is that [Worker.EvaluateHandleAsync(expression, arg)](./api/class-worker.mdx#worker-evaluate-handle) returns [JSHandle].

If the function passed to the [Worker.EvaluateHandleAsync(expression, arg)](./api/class-worker.mdx#worker-evaluate-handle) returns a [Promise], then [Worker.EvaluateHandleAsync(expression, arg)](./api/class-worker.mdx#worker-evaluate-handle) would wait for the promise to resolve and return its value.
