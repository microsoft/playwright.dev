The Worker class represents a [WebWorker](https://developer.mozilla.org/en-US/docs/Web/API/Web_Workers_API). `worker` event is emitted on the page object to signal a worker creation. `close` event is emitted on the worker object when the worker is gone.

```csharp
Page.Worker += (_, worker) =>
{
    Console.WriteLine($"Worker created: {worker.Url}");
    worker.Close += (_, _) => Console.WriteLine($"Worker closed {worker.Url}");
};

Console.WriteLine("Current Workers:");
foreach(var pageWorker in Page.Workers)
{
    Console.WriteLine($"\tWorker: {pageWorker.Url}");
}
```

## event Worker.Close
- type: <[Worker]>

Emitted when this dedicated [WebWorker](https://developer.mozilla.org/en-US/docs/Web/API/Web_Workers_API) is terminated.

## Worker.EvaluateAsync(expression, …)
- `expression` <[string]> JavaScript expression to be evaluated in the browser context. If it looks like a function declaration, it is interpreted as a function. Otherwise, evaluated as an expression.
- `arg` <[EvaluationArgument]> Optional argument to pass to `expression`.
- returns: <[object]>

Returns the return value of `expression`.

If the function passed to the [Worker.EvaluateAsync(expression, …)](./api/class-worker.mdx#workerevaluateasyncexpression-) returns a [Promise], then [Worker.EvaluateAsync(expression, …)](./api/class-worker.mdx#workerevaluateasyncexpression-) would wait for the promise to resolve and return its value.

If the function passed to the [Worker.EvaluateAsync(expression, …)](./api/class-worker.mdx#workerevaluateasyncexpression-) returns a non-[Serializable] value, then [Worker.EvaluateAsync(expression, …)](./api/class-worker.mdx#workerevaluateasyncexpression-) returns `undefined`. Playwright also supports transferring some additional values that are not serializable by `JSON`: `-0`, `NaN`, `Infinity`, `-Infinity`.

## Worker.EvaluateHandleAsync(expression, …)
- `expression` <[string]> JavaScript expression to be evaluated in the browser context. If it looks like a function declaration, it is interpreted as a function. Otherwise, evaluated as an expression.
- `arg` <[EvaluationArgument]> Optional argument to pass to `expression`.
- returns: <[JSHandle]>

Returns the return value of `expression` as a [JSHandle].

The only difference between [Worker.EvaluateAsync(expression, …)](./api/class-worker.mdx#workerevaluateasyncexpression-) and [Worker.EvaluateHandleAsync(expression, …)](./api/class-worker.mdx#workerevaluatehandleasyncexpression-) is that [Worker.EvaluateHandleAsync(expression, …)](./api/class-worker.mdx#workerevaluatehandleasyncexpression-) returns [JSHandle].

If the function passed to the [Worker.EvaluateHandleAsync(expression, …)](./api/class-worker.mdx#workerevaluatehandleasyncexpression-) returns a [Promise], then [Worker.EvaluateHandleAsync(expression, …)](./api/class-worker.mdx#workerevaluatehandleasyncexpression-) would wait for the promise to resolve and return its value.

## Worker.WaitForCloseAsync(…)
- `timeout` <[double]> Maximum time to wait for in milliseconds. Defaults to `30000` (30 seconds). Pass `0` to disable timeout. The default value can be changed by using the [BrowserContext.SetDefaultTimeout(timeout)](./api/class-browsercontext.mdx#browsercontextsetdefaulttimeouttimeout).
- returns: <[Worker]>

Performs action and waits for the Worker to close.
