The Mouse class operates in main-frame CSS pixels relative to the top-left corner of the viewport.

Every `page` object has its own Mouse, accessible with [Page.Mouse](/api/class-page.mdx#page-mouse).

```csharp
await Page.Mouse.MoveAsync(0, 0);
await Page.Mouse.DownAsync();
await Page.Mouse.MoveAsync(0, 100);
await Page.Mouse.MoveAsync(100, 100);
await Page.Mouse.MoveAsync(100, 0);
await Page.Mouse.MoveAsync(0, 0);
await Page.Mouse.UpAsync();
```

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.8</font>

### WheelAsync {#mouse-wheel}

<font size="2" style={{position: "relative", top: "-20px"}}>Added in: v1.15</font>


Dispatches a `wheel` event.

:::note
Wheel events may cause scrolling if they are not handled, and this method does not wait for the scrolling to finish before returning.
:::

**Usage**

```csharp
await Mouse.WheelAsync(deltaX, deltaY);
```

**Parameters**
- `deltaX` &#60;[double]&#62; Pixels to scroll horizontally.<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="mouse-wheel-option-delta-x"/><a href="#mouse-wheel-option-delta-x" class="list-anchor">#</a>
- `deltaY` &#60;[double]&#62; Pixels to scroll vertically.<a aria-hidden="true" tabindex="-1" class="list-anchor-link" id="mouse-wheel-option-delta-y"/><a href="#mouse-wheel-option-delta-y" class="list-anchor">#</a>
